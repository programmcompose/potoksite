# Работа с референсами

Референс — это **готовый трек**, который вы анализируете для обучения. Разбор чужих работ ускоряет прогресс в разы.

## Повторения прогрессии аккордов и мелодии

### Как разобрать прогрессию

1. **Откройте референс** в DAW
2. **Определите тональность** (Newtone, Keyfinder, на слух)
3. **Выделите фразу** (4-8 тактов)
4. **Попробуйте угадать аккорды** — играйте в пиано-ролле
5. **Сравните** с оригиналом

### Пошаговый анализ

```
Шаг 1: Слушайте трек 5-10 раз
Шаг 2: Определите BPM и тональность
Шаг 3: Выпишите аккорды каждого раздела
Шаг 4: Найдите мелодический мотив
Шаг 5: Повторите в своей DAW
```

!!! tip
    Начните с **простых треков** — lo-fi и поп-музыка имеют более очевидные прогрессии, чем экспериментальные жанры.

## Анализ инструментов

### Что анализировать

| Элемент | Вопросы |
|---|---|
| **Мелодия** | Какие ноты? Какой ритм? Какой тембр? |
| **Аккорды** | Какие прогрессии? Какие интервалы? |
| **Бас** | Следует за корнем или независим? |
| **Драмка** | Какой паттерн? Какой свинг? |
| **FX** | Какие эффекты? Где переходы? |
| **Структура** | Как строится трек? Где кульминация? |

### Практический разбор

#### Пример: анализ trap-бита

```
Тональность: C# minor
BPM: 140

[Интро]  0:00-0:16  — Пэд + пианино (Cm9 → Abmaj7)
[Куплет] 0:16-0:48  — 808 + мелодия + драмка
[Припев] 0:48-1:12  — Двойная мелодия, больше FX
[Аутро] 1:12-1:30  — Fade out, реверб
```

### Чек-лист анализа референса

- [ ] Тональность определена
- [ ] BPM найден
- [ ] Аккорды выписаны
- [ ] Мелодический мотив идентифицирован
- [ ] Инструменты распознаны
- [ ] Структура трека понятна
- [ ] Попробовано повторить в DAW

!!! important
    Анализ референсов — не копирование. Это **обучение**. Вы изучаете «рецепт», чтобы готовить свои блюда.
