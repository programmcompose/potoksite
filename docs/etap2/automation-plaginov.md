# Автоматизация плагинов

Автоматизация — изменение параметров плагинов во времени. Превращает статичный микс в живой и динамичный.

## Основные фишки

| Что автоматизировать | Эффект |
|---|---|
| **Громкость (Volume)** | Базовая автоматизация. Элементы «выплывают» когда нужно, отступают когда мешают |
| **Панорама (Pan)** | Движение звука влево-вправо. Создаёт «летающий» эффект |
| **Reverb Send** | Больше реверба на концах фраз, меньше в начале. Динамика пространства |
| **Filter Cut-off** | «Открытие» и «закрытие» фильтра. Классический sweep |
| **Delay Feedback** | Каскад повторений на дропах и переходах |
| **Plugin On/Off** | Включение эффекта только на определённых секциях |

## Применение в структуре трека

| Секция | Типичная автоматизация |
|---|---|
| **Intro** | Постепенное появление элементов, нарастание фильтра |
| **Build-up** | Rising filter, нарастающий delay feedback, увеличение громкости |
| **Drop / Кульминация** | Максимальная ширина, полная открытость фильтра, минимум reverb (чёткость) |
| **Breakdown** | Увеличение reverb, «открытие» пространства, снижение энергии |
| **Outro** | Постепенное затухание, увеличение reverb tail |

!!! tip "Главный принцип"
    Автоматизация — это **рассказ истории**. Трек не должен звучать одинаково от начала до конца. Меняйте параметры, чтобы слушатель «путешествовал».
