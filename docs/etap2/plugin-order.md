# Важность последовательности обработки

Порядок плагинов в цепи (signal chain) определяет итоговый звук. Неправильный порядок = плохой результат, даже если каждый плагин настроен идеально.

## Базовый порядок (от первого к последнему)

| Порядок | Плагин | Почему именно так |
|---|---|---|
| **1** | **EQ (cleanup)** | Убрать мусор (низкие частоты, резонансы) до обработки |
| **2** | **Компрессор** | Работает с «чистым» сигналом. Динамика контролируется до окрашивания |
| **3** | **Сатурация** | Добавляет гармоники после компрессии. Компрессор не «съест» новые частоты |
| **4** | **EQ (tone shaping)** | Коррекция тона после сатурации (гармоники изменили спектр) |
| **5** | **Delay / Reverb** | Пространственные эффекты в конце — они обрамляют уже сформированный звук |

## Что ставить первым

**Всегда начинайте с «уборки»:** High-pass фильтр → срез проблемных частот → потом всё остальное.

Почему: если сначала сатюрировать «грязный» сигнал, вы сатюрируете и грязь.

## Как тренировать понимание

| Метод | Как |
|---|---|
| **A/B сравнение** | Включайте/выключайте всю цепь. Слышите разницу? |
| **Перестановка плагинов** | Поменяйте компрессор и сатурацию местами. Сравните звук |
| **Один плагин за раз** | Добавляйте плагины по одному, слушайте после каждого |

## Ableton vs FL Studio

| DAW | Особенность |
|---|---|
| **Ableton** | Device chain — визуальный порядок. Device Rack для группировки. Return tracks для send-эффектов |
| **FL Studio** | Plugin架 в mixer insert. Sidechain через manual sidechain или ducking. Fruit Send для send-эффектов |

## Сенды и посылы (Send / Return)

Вместо вешания reverb/delay на каждый трек — создайте общий «бус» с эффектом и посылайте на него треки.

| Преимущество | Описание |
|---|---|
| **Один реверб для всех** | Консистентное пространство, меньше CPU |
| **Контроль из одного места** | Изменили reverb — изменился весь микс |
| **Гибкость** | Разные треки — разная сила посылки, один и тот же эффект |
